# 类脑系统 + LLM 混合架构

## 架构设计

这是一个**认知控制层（类脑系统）+ 知识外脑（LLM）**的混合架构：

```
┌─────────────────────────────────────────────────────────────────────┐
│                        类脑认知控制层                                │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐             │
│  │ 感知系统  │→│ 注意系统  │→│ 记忆系统  │→│ 价值系统  │             │

│                                    │           │                    │

│                              ┌─YES─┐          │                    │
│                              ↓     │          ↓                    │
│                         直接响应   │      决策系统                   │
│                              │     │          ↓                    │
│                              │     └────→ 行动选择                  │
│                              │                                     │
│                              ↓ NO                                  │
│                    ┌───────────────────┐                          │
│                    │  检测知识缺口      │                          │
│                    │  • 记忆覆盖度低    │                          │
│                    │  • 感知置信度低    │                          │
│                    └─────────┬─────────┘                          │
│                              ↓                                     │
│                    ┌───────────────────┐                          │
│                    │  加入学习队列      │                          │
│                    │  (异步/低优先级)   │                          │

│                              ↓                                     │
│  ╔═══════════════════════════════════════════════════════════════╗  │
│  ║  实时响应用户（不等LLM）：                                      ║  │

│  ║  基于已有知识给出部分回答                                       ║  │

                                    ↓

│  │  │ 任务3      │  │     │        ↓        │                     │
│  │  └────────────┘  │     │  ┌────────────┐ │                     │
│  │     ↑            │     │  │ LLM生成    │ │                     │
│  │     │            │     │  │ 知识回答   │ │                     │
│  └─────┼────────────┘     │  └─────┬──────┘ │                     │

│        │                  │  知识结构化提取   │                    │
│        │                  │  • 核心概念      │                    │

│        │                  │  • 相关概念      │                    │
```

## 工作流程

### 1. 实时响应流程（毫秒级）

```python
用户提问 → 类脑系统处理（<100ms）
    ↓
1. 感知系统：理解意图，提取关键词
2. 注意系统：聚焦关键信息
3. 记忆检索：查找已有知识
    ↓
┌────────────────────────────────┐
│ 记忆覆盖度 > 阈值？              │
├────────────────────────────────┤
│ YES: 基于记忆生成回答           │
│ NO:  标记知识缺口，加入学习队列  │
│      给出"学习中"或部分回答      │
└────────────────────────────────┘
    ↓
返回给用户（不等待LLM）
```

### 2. 后台学习流程（闲置时执行）

```python
系统闲置（5秒无新请求）
    ↓
从学习队列取出任务
    ↓
构建学习Prompt → 调用LLM API
    ↓
提取结构化知识：
    • 核心概念
    • 关键事实
    • 相关概念
    • 应用场景
    ↓
存入记忆系统：
    • 语义记忆（长期知识）
    • 情景记忆（学习事件）
    ↓
标记为已完成
```

## 核心优势

| 场景 | 传统LLM | 类脑+LLM混合 |
|-----|---------|-------------|
| **响应速度** | 1-10秒 | <100ms（缓存命中）|
| **重复问题** | 每次调用API | 从记忆直接回答 |
| **成本** | 每次请求计费 | 仅新知识调用API |
| **离线使用** | 必须联网 | 已学知识可离线 |
| **个性化** | 无状态 | 记住用户偏好 |
| **可解释性** | 黑盒 | 可追溯知识来源 |

## 使用示例

### 基本用法

```python
from src.brain.orchestrator import get_brain
from src.brain.learning_system import create_simple_llm_client

# 1. 初始化
brain = get_brain()
brain.debug = True

# 2. 设置LLM客户端（可以是OpenAI、Claude、本地模型等）
def my_llm_client(prompt: str) -> str:
    # 调用你的LLM API
    response = call_openai_api(prompt)
    return response

brain.set_llm_client(my_llm_client)

# 3. 启动后台学习
brain.start_background_learning()

# 4. 处理用户请求（实时响应）
response = await brain.process("什么是量子计算？")
print(response.action)  # 行动决策
print(response.metadata["knowledge_gap"])  # 是否有知识缺口

# 5. 查看学习状态
status = brain.get_state_summary()
print(status["learning_system"])
# {
#     "queue_size": 3,          # 待学习队列
#     "learned_count": 5,       # 已学习数
#     "is_idle": True           # 是否闲置
# }

# 6. 停止后台学习
brain.stop_background_learning()
```

### 自定义知识检测阈值

```python
from src.brain.learning_system import KnowledgeLearningSystem

# 创建学习系统时调整参数
learning_system = KnowledgeLearningSystem(
    memory_system=brain.memory,
    llm_client=my_llm_client,
    confidence_threshold=0.3,  # 置信度低于30%触发学习
    max_queue_size=200         # 最多缓存200个学习任务
)
```

### 立即学习（高优先级知识）

```python
# 对于高优先级知识，立即学习，不等待闲置
from src.brain.learning_system import LearningTask, KnowledgeGap

gap = KnowledgeGap(
    topic="重要概念",
    query="什么是...",
    confidence=0.1
)

task = LearningTask(
    query=gap.query,
    context={},
    priority=1  # 高优先级
)

# 同步执行学习（阻塞等待LLM）
success = brain.learning.learn_now(task)
```

## 与纯类脑系统的区别

| 特性 | 纯类脑系统 | 类脑+LLM混合 |
|-----|-----------|-------------|
| **知识来源** | 预定义规则 | LLM实时生成 |
| **知识广度** | 有限 | 无限（LLM覆盖）|
| **响应质量** | 结构化但简单 | 结构化+深度 |
| **API成本** | 0 | 按需付费 |
| **延迟** | 低 | 首问高，后续低 |

## 最佳实践

1. **预热阶段**：启动时预加载常见知识
   ```python
   common_questions = ["你是谁", "你能做什么", ...]
   for q in common_questions:
       brain.learning.learn_now(LearningTask(q, {}, priority=1))
   ```

2. **批量学习**：夜间批量处理学习队列
   ```python
   # 定时任务：每天凌晨处理所有待学习项
   while not brain.learning.learning_queue.empty():
       task = brain.learning.learning_queue.get()
       brain.learning.learn_now(task)
   ```

3. **学习反馈**：根据用户反馈调整学习优先级
   ```python
   if user_feedback == "不满意":
       # 提高相关主题的学习优先级
       task.priority = 2
   ```

## 局限性

1. **首次响应**：新知识首次询问时，只能返回"学习中"或部分回答
2. **知识准确性**：依赖LLM输出的准确性
3. **存储成本**：大量学习会导致记忆系统膨胀（需要遗忘机制）
4. **时效性**：学到的知识不会自动更新（需要定期刷新）

## 适用场景

✅ **适合使用**：
- FAQ问答系统（大量重复问题）
- 客服机器人（特定领域知识）
- 个人助手（记住用户偏好）
- 教育应用（渐进式学习）

❌ **不适合使用**：
- 需要实时信息的场景（天气、股价）
- 创意生成（写作、艺术）
- 单次查询为主的场景

## 总结

这个架构的核心思想是：**用类脑系统做"认知控制"，用LLM做"知识外脑"**。

- **实时路径**：类脑系统独立处理（快速、可解释、有记忆）
- **后台路径**：LLM补充知识（深度、广度、准确）

就像人脑一样：
- 你学过的知识，下次可以直接回忆（记忆命中）
- 不会的知识，去查书/百度（LLM查询）
- 查到后记住，下次就不用再查了（学习存储）
