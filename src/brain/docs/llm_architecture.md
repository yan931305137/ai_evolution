# 大语言模型 (LLM) 架构详解

## 核心架构：Transformer

当前主流LLM（GPT、Claude、Llama等）都基于 **Transformer** 架构，2017年由Google提出。

```
┌─────────────────────────────────────────────────────────────────┐
│                         LLM 整体架构                            │
├─────────────────────────────────────────────────────────────────┤
│  输入文本                                                         │
│     ↓                                                           │
│  Tokenizer（分词器）→ Token IDs                                  │
│     ↓                                                           │
│  Embedding层（词向量 + 位置编码）                                  │
│     ↓                                                           │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │           Transformer Block × N（24-96层）              │    │
│  │  ┌─────────────────────────────────────────────────┐    │    │
│  │  │  Multi-Head Self-Attention（多头自注意力）       │    │    │
│  │  │  • 计算每个token与其他所有token的关系              │    │    │
│  │  │  • 权重 = Query · Key / √d                       │    │    │
│  │  │  • 复杂度 O(n²)                                  │    │    │
│  │  └─────────────────────────────────────────────────┘    │    │
│  │                          ↓                             │    │
│  │  ┌─────────────────────────────────────────────────┐    │    │
│  │  │  Feed-Forward Network（前馈神经网络）            │    │    │
│  │  │  • 两个线性变换 + 激活函数                        │    │    │
│  │  │  • 扩大模型容量                                   │    │    │
│  │  └─────────────────────────────────────────────────┘    │    │
│  │                          ↓                             │    │
│  │  Layer Norm + Residual Connection（层归一化+残差）      │    │
│  └─────────────────────────────────────────────────────────┘    │
│     ↓                                                           │
│  Output Head（输出头）→ 预测下一个token的概率分布                  │
│     ↓                                                           │
│  采样生成 → 输出文本                                              │
└─────────────────────────────────────────────────────────────────┘
```

## 核心组件详解

### 1. Tokenizer（分词器）

```python
# 示例：文本如何变成数字
输入: "Hello world"
     ↓
Tokenizer
     ↓
Tokens: ["Hello", " world"]  或子词 ["He", "ll", "o", " world"]
     ↓
Token IDs: [15496, 995]  ← 词汇表中的索引
```

- 将文本切分为token（词或子词）
- 建立词汇表（通常5万-10万个token）
- 转换为模型可处理的整数序列

### 2. Embedding（嵌入层）

```
Token ID: 15496
     ↓
Embedding Table（查表）
     ↓
向量: [0.23, -0.56, 0.89, ..., 0.12]  ← 通常512-4096维

+ 位置编码（告诉模型token的位置）
     ↓
输入向量序列
```

- 将离散token映射为连续向量
- 添加位置信息（positional encoding）

### 3. Self-Attention（自注意力）- 核心机制

```
对于序列中的每个token:
                     ┌─────────────────────────┐
输入向量 ──→ [Wq] ──→│ Query (查询)             │
             [Wk] ──→│ Key (键)                │ ←── 所有token
             [Wv] ──→│ Value (值)              │
                     └─────────────────────────┘
                              ↓
              Attention = softmax(Q · Kᵀ / √d) · V
                              ↓
                     加权求和输出
```

**计算过程示例：**
```
句子: "猫 坐 在 垫子 上"

对于"猫"这个token:
          Q_cat · K_猫   Q_cat · K_坐   Q_cat · K_在   Q_cat · K_垫子  Q_cat · K_上
注意力权重 = [    0.4    ,    0.2    ,    0.1    ,     0.2     ,    0.1    ]
                    ↓
输出 = 0.4×V_猫 + 0.2×V_坐 + 0.1×V_在 + 0.2×V_垫子 + 0.1×V_上
```

- **每个token都能看到所有其他token**
- 并行计算所有位置的注意力
- 复杂度：O(n²)，n = 序列长度

### 4. Multi-Head Attention（多头注意力）

```
输入 ──→ [Head 1] ──┐
         [Head 2] ──┤
         [Head 3] ──┼──→ Concat ──→ Linear ──→ 输出
         ...       ──┤
         [Head H] ──┘

通常 H = 8-96 个头，每个头学习不同的注意力模式
```

### 5. Feed-Forward Network（前馈网络）

```
输入 ──→ Linear ──→ Activation(GeLU/ReLU) ──→ Linear ──→ 输出
         (4d → d)                              (d → d)
         
作用：增加非线性变换能力，扩大模型容量
```

### 6. Layer Normalization + Residual

```
输入 ──→ SubLayer ──→ + ──→ 输出
         │            ↑
         └────────────┘ (残差连接，解决梯度消失)
         ↓
       LayerNorm (层归一化，稳定训练)
```

## 训练过程

### 阶段1: 预训练 (Pre-training)

```
海量文本数据（互联网、书籍、代码等）
            ↓
    自监督学习：预测下一个token
            ↓
输入: "今天天气很" → 模型预测 → "好" (目标)
            ↓
    计算交叉熵损失 → 反向传播 → 更新参数
            ↓
    重复数万亿次（数周-数月，数千GPU）
```

**目标函数：**
```
Loss = -Σ log P(x_i | x_1, x_2, ..., x_{i-1})
```

### 阶段2: 微调/对齐 (Fine-tuning/RLHF)

```
预训练模型（基础能力）
         ↓
┌─────────────────────────────────────────┐
│  SFT (监督微调)                          │
│  用高质量对话数据继续训练                  │
└─────────────────────────────────────────┘
         ↓
┌─────────────────────────────────────────┐
│  RLHF (人类反馈强化学习)                  │
│  1. 训练奖励模型（人类偏好）              │
│  2. 用PPO算法优化策略                    │
└─────────────────────────────────────────┘
         ↓
    对齐后的对话模型
```

## 推理过程（生成文本）

```
用户输入: "请介绍一下人工智能"
              ↓
    Tokenize → [101, 234, 567, ...]
              ↓
    输入模型
              ↓
    输出概率分布: P(下一个token | 输入)
    例如: "人": 0.3, "的": 0.2, "技术": 0.15, ...
              ↓
    采样策略（Greedy / Temperature / Top-p）
              ↓
    选择token: "人"
              ↓
    拼接到输入，重复直到生成结束符
              ↓
    最终输出: "人工智能是一种模拟人类智能的技术..."
```

## LLM的本质特点

### 1. **统计模式匹配器**
```
不是"理解"，而是学习训练数据中的统计规律
输入 → 寻找最可能出现的下一个token组合
```

### 2. **无状态（ Stateless ）**
```
每次推理都是独立的
没有真正的"记忆"，只有固定的上下文窗口
长对话需要把历史记录全部传入
```

### 3. **无法持续学习**
```
训练完成后参数固定
新信息只能通过prompt注入（上下文学习）
无法从交互中真正"学习"
```

### 4. **计算密集**
```
推理需要矩阵乘法: Y = X · W
参数越多，计算量越大
需要GPU加速
```

## LLM vs 类脑系统对比

| 维度 | 传统LLM | 类脑系统 |
|-----|---------|---------|
| **架构** | 统一Transformer | 分区专业化系统 |
| **注意力** | 全局O(n²) | 选择性O(k) |
| **记忆** | 上下文窗口 | 多层记忆+巩固 |
| **学习** | 离线预训练 | 在线持续学习 |
| **决策** | 概率采样 | 价值驱动决策 |
| **参数** | 十亿-万亿级 | 无参数（规则驱动） |
| **计算** | GPU必需 | CPU即可 |
| **解释性** | 黑盒 | 系统级透明 |

## 关键问题

### 为什么LLM会产生幻觉？
```
原因：模型学习的是"什么样的回答像训练数据"
      而不是"什么样的回答符合事实"
      
生成: "爱因斯坦获得诺贝尔文学奖" 
      ↓
因为训练数据中有"爱因斯坦获得...奖"的模式
模型没有事实验证机制
```

### 为什么LLM上下文有限？
```
原因：注意力机制O(n²)复杂度
      
上下文4K → 计算量 16M
上下文128K → 计算量 16B (增加1000倍)

内存和计算成本急剧上升
```

### 为什么LLM无法真正推理？
```
原因：所有"推理"都是训练数据中模式的组合
      没有显式的逻辑规则引擎
      
      对于没见过的新问题，容易失败
      缺乏像人脑那样的工作记忆和规划能力
```

## 总结

**LLM本质**：一个超大规模的**统计语言模型**
- 输入：token序列
- 处理：多层注意力变换
- 输出：下一个token的概率分布

**核心局限**：
1. 无状态、无记忆
2. 无法持续学习
3. 黑盒决策
4. 计算成本高

**这正是类脑系统试图解决的问题** —— 通过模拟人脑的分区结构和认知机制，实现真正的持续学习、选择性注意和价值驱动决策。
