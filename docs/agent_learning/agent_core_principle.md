# Agent核心原理与四大模块协作逻辑学习笔记

## 一、四大核心模块定义
### 1. 感知模块（Perception）
- 功能：负责获取外部环境的所有输入信息，是Agent与外界交互的入口
- 输入来源：用户指令、工具返回结果、环境状态变化、多模态数据（文本/语音/图像等）
- 处理逻辑：对输入信息做标准化格式转换、过滤无效信息、提取关键特征

### 2. 记忆模块（Memory）
- 分类：
  - 短期记忆（工作记忆）：当前会话上下文、任务执行中间状态，存储在会话生命周期内，容量有限
  - 长期记忆：历史交互记录、知识库、工具调用规则、过往任务成功/失败经验，持久化存储，容量大
- 功能：为规划模块提供上下文参考，避免重复执行相同操作，支持经验复用

### 3. 规划模块（Planning）
- 功能：Agent的大脑决策中心，基于感知输入+记忆内容，完成任务拆解、路径制定、优先级排序
- 常见实现：思维链（CoT）、思维树（ToT）、反射（Reflection）、子任务拆分逻辑
- 核心逻辑：判断当前任务是否需要调用工具、是否需要拆分步骤、是否完成当前阶段任务

### 4. 行动模块（Action）
- 功能：执行规划模块输出的决策指令，包括工具调用、结果输出、状态更新
- 常见动作：信息检索、数据计算/处理、外部API调用、生成回答反馈给用户
- 反馈逻辑：执行结果会返回给感知模块，同时更新记忆模块内容，形成执行闭环

## 二、四大模块协作完整链路
1. 用户输入指令 → 感知模块接收并处理输入
2. 感知模块将处理后的信息传入规划模块，同时记忆模块检索相关历史信息和知识库内容提供给规划模块
3. 规划模块根据输入+记忆内容，判断任务类型，制定执行计划
   - 如果任务需要调用工具：输出工具调用指令给行动模块
   - 如果任务无需工具可直接回答：直接生成结果返回给用户
4. 行动模块执行工具调用/任务处理，将执行结果返回给感知模块，同时将执行过程和结果写入记忆模块
5. 感知模块将行动结果传入规划模块，判断当前任务是否完成：
   - 未完成：重复步骤3-5，迭代执行直到任务完成
   - 已完成：整理最终结果反馈给用户

## 三、后续待复现三类场景示例说明
1. 信息检索场景：Agent接收用户问题，调用搜索工具获取信息，整合后返回答案
2. 数据处理场景：Agent接收结构化/非结构化数据，完成清洗、计算、分析等操作，输出处理结果
3. 工具调用场景：Agent根据任务需求，自主选择合适的工具（计算器/天气查询/文件操作等）完成任务
