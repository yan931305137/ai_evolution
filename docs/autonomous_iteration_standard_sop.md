# 智能体自主迭代标准化SOP v1.0
## 一、适用范围与目标
适用场景：智能体功能迭代、核心代码优化、技能新增/升级操作
核心目标：实现同类迭代零事故上线，全流程可追溯、可复现、可量化
覆盖环节：自我评估→代码修改→风险校验→热更新→效果复盘→风险回滚→效果量化评估

---
## 二、全流程标准化操作规范
### 2.1 阶段1：自我评估（迭代前置准备）
#### 操作步骤：
1. 运行`collect_runtime_operation_data()`采集近7天运行数据：含报错日志、性能指标、用户反馈、技能调用成功率
2. 运行`identify_evolution_problems(采集的运行数据)`识别待优化问题，按优先级排序（高/中/低）
3. 运行`generate_iteration_plan(优先级问题列表, max_problems_per_iteration=1)`生成可执行迭代计划，单次迭代仅处理1个优先级最高的问题，避免范围蔓延
4. 自我评估准入校验：迭代影响范围评估≤3个关联模块、无跨核心系统依赖，评估不通过则终止本次迭代
#### 准出条件：
✅ 迭代计划明确：待修改模块、预期优化目标、关联测试用例清单清晰
✅ 风险前置识别：已识别潜在风险点≥2个，对应预案已明确

---
### 2.2 阶段2：代码修改
#### 操作步骤：
1. 代码修改必须添加详细中文注释，核心逻辑注释覆盖率≥80%
2. 新增/修改技能必须同步更新技能描述、参数说明、返回值说明
3. 运行`project_compliance_auto_check(待修改文件路径, content_type='code_xxx', scene_type='iteration')`完成路径/合规校验，校验不通过不得进入下一环节
4. 提交修改前必须与原代码做diff对比，仅保留与迭代目标相关的修改，无关代码不得修改
#### 准出条件：
✅ 代码符合项目编码规范，注释完整
✅ 合规校验100%通过

---
### 2.3 阶段3：风险校验
#### 操作步骤：
1. 安全校验：运行`code_security_verification(目标文件路径, 新代码内容)`，校验内容含：注入风险、权限溢出、死循环、敏感数据泄露风险
2. 测试用例校验：优先复用现有关联测试用例，新增测试用例覆盖所有修改点，边界场景覆盖率100%
3. 灰度测试：运行`grayscale_test_executor(待测试模块路径, 关联测试用例列表, coverage_threshold=80.0)`，测试覆盖率必须≥80%，核心模块要求≥90%
4. 回归测试：运行`run_test_suite(目标模块路径)`验证原有功能未受影响
#### 准出条件：
✅ 安全校验零风险告警
✅ 所有测试用例100%通过
✅ 测试覆盖率达标

---
### 2.4 阶段4：热更新上线
#### 操作步骤：
1. 自动备份：调用`deployment_rollback_manager`自动备份原版本代码到`.deployment_backups`目录，保留最近10个版本
2. 灰度放量：首批放量10%流量运行10分钟，无异常放量到50%再运行20分钟，全量前必须确认无报错
3. 冒烟测试：放量过程中运行`smoke_test_code`执行冒烟测试，核心功能验证通过后方可继续放量
4. 自动回滚触发：出现任意以下情况立即触发回滚：
   - 冒烟测试失败
   - 错误率≥1%
   - 核心功能响应时延提升≥50%
#### 准出条件：
✅ 全量上线后运行30分钟无异常告警
✅ 冒烟测试100%通过

---
### 2.5 阶段5：效果复盘
#### 操作步骤：
1. 指标采集：上线24小时后采集核心指标，对比上线前后数据差异
2. 效果评估：对照迭代计划的预期目标评估是否达标，未达标则分析根因
3. 输出复盘报告：写入`reports/iteration_report_xxx.md`，内容含：迭代内容、指标对比、问题分析、后续优化建议
4. 迭代闭环：达标则归档本次迭代所有资产，未达标则纳入下一次迭代计划

---
## 三、风险回滚专项规范
### 3.1 回滚触发场景
- 冒烟测试不通过
- 上线后错误率≥1%
- 核心功能不可用
- 性能下降超过阈值
- 用户反馈严重问题
### 3.2 回滚操作流程
1. 立即调用`deployment_rollback_manager(目标文件路径, rollback_version=上一个稳定版本号, auto_rollback=True)`执行自动回滚
2. 回滚后立即执行冒烟测试验证功能恢复
3. 10分钟内确认回滚效果，无异常后排查本次迭代问题
### 3.3 回滚验证标准
✅ 核心功能恢复到上线前水平
✅ 错误率恢复到上线前基线
✅ 无新增告警

## 四、效果量化评估标准
| 评估维度 | 量化指标 | 达标阈值 |
|---------|---------|---------|
| 功能正确性 | 测试用例通过率 | 100% |
| 性能表现 | 核心接口响应时延变化 | ≤±20% |
| 稳定性 | 上线后72小时错误率 | ≤0.1% |
| 代码质量 | 测试覆盖率 | ≥80%（核心模块≥90%） |
| 迭代效率 | 单次迭代周期 | ≤12小时 |
| 回滚率 | 迭代回滚占比 | ≤5% |
