# 信息获取渠道配置模块使用说明

## 概述

配置模块提供了完整的业务配置管理功能，支持分类标签、筛选标准、清理规则等业务配置的统一管理和便捷访问。

## 配置文件位置

- 主配置文件：`src/config/config.yaml`
- 配置管理工具：`src/utils/channel_config.py`

## 功能模块

### 1. 分类标签配置

支持5种信息源分类：

- **官方权威**（优先级1）：政府机构、官方组织发布的权威信息源
- **专业媒体**（优先级2）：专业媒体、期刊、研究报告等
- **行业资讯**（优先级3）：行业动态、市场资讯类信息源
- **社交媒体**（优先级4）：社交媒体平台、论坛等用户生成内容
- **个人博客**（优先级5）：个人博客、自媒体等

### 2. 自动分类规则

基于关键词匹配的自动分类功能：

```python
from src.utils.channel_config import channel_config

# 自动分类
category = channel_config.classify_by_keywords(
    url="http://www.gov.cn/news",
    title="政府公告"
)
# 返回: "官方权威"
```

### 3. 筛选标准配置

#### 3.1 多维度评分

- **权威性评分**（0-100）
  - 官方来源权重：30
  - 专业媒体权重：25
  - 行业机构权重：20
  - 个人来源权重：10
  - 未知来源权重：5

- **稳定性评分**（0-100）
  - 更新频率权重：40
  - 可用性权重：30
  - 历史可靠性权重：30

- **内容质量评分**（0-100）
  - 专业深度权重：35
  - 信息密度权重：30
  - 准确性验证权重：20
  - 可读性权重：15

#### 3.2 筛选阈值

- 最低权威性分数：60
- 最低稳定性分数：50
- 最低内容质量分数：55
- 最低综合分数：65
- 最大广告占比：25%

#### 3.3 综合评分计算

```python
from src.utils.channel_config import channel_config

# 计算综合评分
overall = channel_config.calculate_overall_score(
    authority=85,
    stability=70,
    content=80
)
# 返回: 78.33

# 判断是否为优质源
is_high_quality = channel_config.is_high_quality_source({
    "authority": 85,
    "stability": 70,
    "content": 80,
    "overall": 78.33
})
# 返回: True
```

### 4. 广告识别配置

- 广告密度阈值：30%
- 疑似关键词：["广告", "赞助", "推广", "sponsored", "ad"]
- 疑似域名：["doubleclick.net", "googleadservices.com"]

### 5. 清理规则配置

#### 5.1 低质源识别规则

- 长期未更新：超过90天未更新
- 广告占比：超过40%广告内容
- 质量评分：低于30分
- 错误率：超过50%

#### 5.2 自动清理策略

- 启用状态：已启用
- 清理间隔：7天
- 批量大小：100
- 试运行模式：关闭

#### 5.3 清理判断

```python
from src.utils.channel_config import channel_config

# 判断是否应该清理
should_cleanup, reason = channel_config.should_cleanup_source({
    "days_since_last_update": 100,
    "ad_ratio": 0.2,
    "quality_score": 75,
    "error_rate": 0.1
})
# 返回: (True, "超过90天未更新（100天）")
```

### 6. 统计分析配置

#### 6.1 有效性指标

- valid_info_ratio：有效信息占比
- update_frequency：更新频率
- quality_score：质量评分
- ad_ratio：广告占比
- error_rate：错误率

#### 6.2 报告生成

- 报告类型：日汇总、周趋势、月分析
- 导出格式：JSON、CSV、Markdown
- 生成间隔：7天

## 使用示例

### 示例1：完整的分类和筛选流程

```python
from src.utils.channel_config import channel_config

# 1. 自动分类信息源
category = channel_config.classify_by_keywords(
    url="https://news.industry.com/report",
    title="行业报告"
)

# 2. 评估信息源质量
scores = {
    "authority": 75,
    "stability": 68,
    "content": 72
}
overall = channel_config.calculate_overall_score(**scores)
scores["overall"] = overall

# 3. 判断是否为优质源
is_quality = channel_config.is_high_quality_source(scores)

# 4. 检查是否需要清理
should_cleanup, reason = channel_config.should_cleanup_source({
    "days_since_last_update": 45,
    "ad_ratio": 0.18,
    "quality_score": overall,
    "error_rate": 0.05
})

print(f"分类: {category}")
print(f"综合评分: {overall}")
print(f"是否优质: {is_quality}")
print(f"是否需要清理: {should_cleanup} ({reason})")
```

## 配置修改

### 修改分类标签

编辑 `src/config/config.yaml`：

```yaml
classification:
  categories:
    - name: "官方权威"
      description: "政府机构、官方组织发布的权威信息源"
      priority: 1
    # 添加新的分类
    - name: "新分类"
      description: "分类描述"
      priority: 6
```

### 修改筛选阈值

```yaml
screening_criteria:
  quality_source_threshold:
    min_authority_score: 70  # 修改阈值
    min_stability_score: 60
    min_content_score: 65
    min_overall_score: 75
    max_ad_ratio: 0.20
```

### 修改清理规则

```yaml
cleanup_rules:
  auto_cleanup:
    enabled: true
    cleanup_interval_days: 3  # 修改清理间隔
    batch_size: 50
    dry_run_mode: true  # 开启试运行模式
```

## API 参考

- `should_cleanup_source(source_data)`: 判断是否应该清理

- `get_analytics_config()`: 获取统计分析配置
- `get_effectiveness_metrics()`: 获取有效性指标列表
- `validate_config()`: 验证配置完整性

## 注意事项

1. **配置热加载**：修改配置文件后需要重启应用或重新加载配置
2. **阈值调整**：筛选阈值的调整会影响优质源的识别结果，建议谨慎调整
3. **试运行模式**：首次启用自动清理时，建议先开启试运行模式（dry_run_mode: true）
4. **配置备份**：重要配置修改前建议备份原有配置文件

## 常见问题

### Q: 如何添加新的分类标签？

A: 编辑 `config.yaml` 中的 `classification.categories` 列表，添加新的分类项，并在 `auto_classification_rules` 中添加对应的分类规则。

### Q: 如何调整筛选严格度？

A: 修改 `screening_criteria.quality_source_threshold` 中的各项阈值，阈值越高筛选越严格。

### Q: 如何测试新配置？

A: 运行 `test_channel_config.py` 测试脚本，确保所有配置项正常工作后再部署。
