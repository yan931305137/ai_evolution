# 任务执行记录自动复盘工具使用说明

## 一、工具概述
### 1.1 功能定位
自动拉取指定时间范围内所有任务执行日志，统计工具调用成功率、失败场景、耗时分布，自动识别高频工具缺口，帮助快速定位系统能力短板，输出工具需求优先级排名。

### 1.2 预期效率提升
- 人工复盘时间从2天/次降低到10分钟/次，效率提升95%
- 工具缺口识别准确率达90%+

## 二、调用方式
### 2.1 函数接口
python
def task_execution_review(
    start_time: Optional[str] = None, 
    end_time: Optional[str] = None, 
    task_type: Optional[str] = None,
    top_n: int = 10,
    log_data_path: str = "./data/near_3month_runtime_data.json"
) -> Dict


### 2.2 参数说明
| 参数名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| start_time | str | 否 | 3个月前 | 统计开始时间，格式：YYYY-MM-DD HH:MM:SS |
| end_time | str | 否 | 当前时间 | 统计结束时间，格式：YYYY-MM-DD HH:MM:SS |
| task_type | str | 否 | 全部 | 任务类型过滤，例如："General Chat"/"Tool Call" |
| top_n | int | 否 | 10 | 高频缺口场景返回数量 |
| log_data_path | str | 否 | 近3个月运行数据文件 | 日志数据文件路径 |

### 2.3 返回结果说明

{
    "status": "success/failed",
    "message": "执行结果说明",
    "summary": {
        "time_range": {"start": "开始时间", "end": "结束时间"},
        "total_records": 总记录数,
        "task_type_filter": "过滤的任务类型"
    },
    "tool_usage_stats": {
        "total_calls": 总调用次数,
        "success_calls": 成功调用次数,
        "failed_calls": 失败调用次数,
        "success_rate": 成功率(%),
        "failed_scenarios": {"失败场景": 出现次数},
        "tool_gap_scenarios": {"缺口场景": 出现次数}
    },
    "time_cost_distribution": {"耗时区间": 数量},
    "top_gap_scenarios": [Top N缺口场景列表],
    "full_priority_ranking": [完整优先级排名列表]
}


### 2.4 调用示例
python
from src.tools.task_review_tool import task_execution_review

# 示例1：默认参数调用，复盘近3个月所有数据
report = task_execution_review()
print(report)

# 示例2：自定义时间范围和任务类型
report = task_execution_review(
    start_time="2026-02-01 00:00:00",
    end_time="2026-02-28 23:59:59",
    task_type="Tool Call",
    top_n=5
)
print(report)


## 三、异常处理方案
### 3.1 常见异常场景与解决方案

| 异常类型 | 触发场景 | 解决方案 |
|----------|----------|----------|
| FileNotFoundError | 日志数据文件不存在 | 检查log_data_path参数是否正确，确认文件存在 |
| 时间格式错误 | start_time/end_time参数不符合YYYY-MM-DD HH:MM:SS格式 | 修正时间参数格式，使用正确的时间字符串 |
| 无匹配记录 | 指定时间范围或任务类型下没有符合条件的记录 | 扩大时间范围或调整任务类型过滤条件 |
| JSON解析错误 | 日志数据文件不是合法的JSON格式 | 检查日志文件内容，确保是合法JSON结构 |

### 3.2 异常返回示例

{
    "status": "failed",
    "message": "复盘分析失败: time data '2026/02/01' does not match format '%Y-%m-%d %H:%M:%S'",
    "data": {}
}


### 3.3 注意事项
1. 日志数据文件必须是符合预设结构的JSON格式，包含recent_interactions字段，每条记录需要有timestamp、input、output、instruction字段
2. 耗时分布当前为模拟实现，后续可根据实际日志中的耗时字段替换为真实统计
3. 工具缺口识别基于关键字匹配，可根据实际场景调整gap_keywords和failed_keywords列表优化识别准确率
