## 二、功能模块划分方案
### 1. 目录结构统一规范
| 目录 | 功能说明 |
|------|----------|
| `src/` | 唯一主源码目录，所有业务代码、核心模块、工具集都存放于此 |
| `src/agents/` | 智能体核心调度逻辑 |
| `src/business/` | 业务场景功能实现 |
| `src/core/` | 核心通用能力模块 |
| `src/tools/` | 通用工具集（文件、代码、仓库、多媒体等） |
| `src/utils/` | 通用工具函数 |
| `src/storage/` | 数据存储相关模块 |
| `src/config/` | 统一配置管理 |
| `src/tests/` | 单元测试、集成测试用例 |
| `docs/` | 项目文档、架构设计、操作手册 |
| `scripts/` | 独立运维、部署、初始化脚本 |
| `data/` | 运行数据、静态资源 |
| `assets/` | 多媒体资源、静态文件 |

### 2. 模块职责划分
- **接入层**：提供多种用户交互入口，统一请求校验、流量管控
- **业务调度层**：负责任务编排、工作流管理、权限管控，隔离接入层和核心能力
- **核心能力层**：实现产品核心业务能力，是产品价值核心载体
- **工具服务层**：通用可复用工具能力，支持所有上层业务调用
- **数据存储层**：统一数据持久化、缓存、检索能力，提供标准化数据访问接口
