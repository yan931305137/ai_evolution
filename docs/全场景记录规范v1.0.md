# AI智能体全场景记录规范 v1.0

## 一、适用范围
本规范适用于智能体自我意识更新模块、经验记忆模块的所有运行记录，覆盖所有成功/异常运行场景，为问题排查、自我迭代、性能优化提供统一数据标准。

## 二、原有记录规则梳理
### 2.1 自我意识更新模块原有规则
1. 记录内容：能力使用记录（名称、成功率、使用次数、信心度）、限制认知记录、反思记录（主题、内容、洞察、时间戳、情感状态）、自我评估历史、成就记录
2. 存储方式：内存对象存储，最多保留50条反思记录
3. 缺失点：无异常场景专门记录、无统一日志格式、无持久化存储保障

### 2.2 经验记忆模块原有规则
1. 记录内容：对话记忆、知识记忆，每条记录包含文档内容、元数据（重要度、过期时间、创建时间）、向量嵌入
2. 存储方式：ChromaDB向量数据库持久化存储
3. 缺失点：异常场景记录字段不全、无统一的执行结果标记、失败场景分类缺失

## 三、全场景分类
### 3.1 成功场景
- 自我意识更新成功（能力更新、认知层级提升、反思生成完成）
- 经验记忆操作成功（写入、查询、更新、删除、维护任务执行完成）

### 3.2 异常场景分类
| 分类ID | 分类名称 | 描述 | 包含子场景 |
|--------|----------|------|------------|
| E01 | 超时类 | 执行时间超过预设阈值导致失败 | 1. 自我意识更新超时<br>2. 记忆写入/查询超时<br>3. 记忆维护任务超时<br>4. 向量嵌入生成超时 |
| E02 | 执行中断类 | 执行过程被外部/内部信号中断导致失败 | 1. 用户主动终止<br>2. 系统重启中断<br>3. 依赖模块故障中断<br>4. 优先级更高的任务抢占中断 |
| E03 | 资源耗尽类 | 系统资源不足导致执行失败 | 1. 内存不足（OOM）<br>2. 磁盘空间不足<br>3. 连接池耗尽<br>4. CPU占用率超过阈值 |
| E04 | 业务异常类 | 业务逻辑校验不通过导致失败 | 1. 记忆内容违规（合规校验不通过）<br>2. 自我认知数据格式异常<br>3. 重复提交请求<br>4. 权限不足 |
| E05 | 未知异常类 | 未归类的其他异常 | 所有未匹配上述分类的失败场景 |

## 四、必填记录字段规范
### 4.1 公共必填字段（所有场景均需包含）
| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| record_id | string | 全局唯一记录ID | uuid字符串 |
| scene_type | string | 场景类型：success/error | success |
| module | string | 所属模块：self_awareness/memory | self_awareness |
| operation | string | 操作类型 | capability_update / memory_write |
| timestamp | int | 操作时间戳（毫秒级） | 1740000000000 |
| cost_time | int | 耗时（毫秒） | 234 |
| operator | string | 触发者：system/user/scheduled_task | scheduled_task |
| trace_id | string | 调用链路ID，用于链路追踪 | trace_abc123 |

### 4.2 成功场景特有字段
| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| result_data | dict | 执行结果数据 | {"capability": "reasoning", "success_rate": 0.85} |
| change_log | list | 变更内容列表 | ["能力推理信心度从80提升到85"] |

### 4.3 异常场景特有字段
| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| error_code | string | 异常分类编码 | E0101 |
| error_msg | string | 异常详细描述 | 自我意识更新超时，预设阈值1000ms，实际耗时1500ms |
| stack_trace | string | 异常堆栈信息 | 完整堆栈字符串 |
| resource_snapshot | dict | 异常发生时的资源快照 | {"cpu_usage": 95, "memory_usage": 92, "disk_usage": 87} |
| retryable | bool | 是否可重试 | true |

## 五、存储格式规范
1. 统一使用JSON格式存储，编码为UTF-8
2. 存储介质：
   - 热数据（30天内）：同时存储到本地日志文件+向量数据库元数据字段
   - 冷数据（超过30天）：定期归档到对象存储/本地压缩文件
3. 日志文件命名规则：{module}_{date}.log，例如 self_awareness_20260228.log
4. 索引规则：按trace_id、timestamp、module、scene_type建立索引，保证查询效率

## 六、评审校验标准
本规范通过标准：
1. 覆盖所有核心失败场景（超时、执行中断、资源耗尽三类均无遗漏）
2. 所有必填字段满足问题排查、性能优化、自我迭代的数据分析需求
3. 存储格式兼容现有系统架构，无额外高额改造代价
